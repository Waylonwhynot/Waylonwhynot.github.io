GitHub搜索指令

1. in：关键词检索

  • 关键词:

    • name: 项目名称
    • description：描述
    • readme： 帮助文档

  • 语法:

​ 需要检索的内容 in:name或description或readme

  • 单独使用:
    • (1) 例如：检索项目名称包含vue的项目
      • Vue in:name
    • (2) 检索项目描述包含vue的项目
      • Vue in:description
    • (3) 检索帮助文档中包含vue的项目
      • Vue in:readme
  • 组合使用:
    • Vue in:readme,description

2. star 或 fork关键词查找

  • 单独语法
    • 数量范围：xxx关键词 stars:>=或者:<=
      • vue stars:>=2000
    • 区间范围: xxx关键词 fork：数字1..数字2
      • springboot fork:500..2000
  • 组合使用
    • redis stars:500..2000 fork:200..2000

3.高亮代码

  • 高亮显示某一行
    • 项目URL#L要显示的代码
    • 例如: 项目URL#L20-L40

4.搜索学习资料 awesome

  • 语法：awesome关键字
  • 查询Python学习资料
    • awesome python

5.交友 location和language

  • 语法: location:地区 language:语言
  • 查找上海地区的go语言使用者
    • location:shanghai language:go

2.pip freeze 导出含有路径 (@ FILE:///) 问题小记

问题

在 conda 沙箱环境中使用 pip freeze > requirements.txt 命令导出已安装的模块，其中部分模块显示了 @ file:///...，而不是具体的版本号，如下图：


此时，如果我们直接在其他机器上边使用 pip install -r requirements.txt 安装模块时，就会遇到如下错误：

原因

这是 pip 安装软件包的一种特殊语法（自19.1开始受支持）PEP404，
但是该此种路径取决于环境，file:///URL 仅在本地文件系统上可用，你不能将生成的 requirements.txt 文件提供给其他人使用
