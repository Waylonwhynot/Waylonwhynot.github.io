一. ansible基本常识

ansible 命令使用如下\

ansible [组或主机] -m [模块] -a [模块参数]

-D, –diff 可选；修改文件时，会输出修改内容

-C, –check 可选；使用该参数时，不会做修改，只是输出可能的修改

-e, –extra-vars 可选；输出额外参数

-f FORKS, –forks=FORKS 可选；指定进程数

-i INVENTORY, –inventory 可选；不使用默认的 inventory 文件，手动指定

-m 指定一个模块

–private-key, –key-file 指定 ssh 连接的 key 文件

-u 指定 ssh 连接使用的用户名

-s 远程登录用户名名非 root，我们需要 sudo 切换

–list-hosts 查看一个组的主机列表

大家可以使用 ansible --help 看具体信息

eg:

ansible-playbook 命令使用如下\

ansible-playbook [playbook 文件]

–flush-cache 清除缓存

–force-handlers 即使出现错误，也会忽略

–list-tasks 查看 playbook 文件的任务列表

更多信息可使用 ansible-playbook --help 查看

eg: 新建一个 playbook 路径为 playbooks/test.yml

ansible-console 命令使用\

ansible-console [组或主机] # 进入一个交互式 shell，可对多台主机执行命令

该命令了解即可，大家在工作中可能用到，项目中并不会使用

6. fetch 模块\

fetch 模块用来拉取远程服务器的文件，参数包括，dest 拉取文件后存储位置；src 目标机器的文件路径；flat 是否覆盖件完整路径。

如果多台服务器，且 flat 参数为 yes 可使用 dest: /tmp/prefix-{{inventory_hostname}} 区分文件。本机生成文件路径 /tmp/prefix-localhost。\
