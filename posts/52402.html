Etcd

1. etcd运⾏机制

etcd是CoreOS团队于2013年6⽉发起的开源项⽬，它的⽬标是构建⼀个⾼可⽤的分布式键值(key-value)数据库。etcd内部采⽤raft协议作为⼀致性算法，etcd基于Go语⾔实现。

https://etcd.io/docs/v3.4/op-guide/hardware/ #官⽅硬件推荐

官方网站

官⽅⽹站：https://etcd.io/

github地址：https://github.com/etcd-io/etcd

etcd有多个不同的API访问版本，v1版本已经废弃，etcd v2 和 v3 本质上是共享同⼀套 raft 协议代码的两个独⽴的应⽤，接⼝不⼀样，存储不⼀样，数据互相隔离。也就是说如果从 Etcd v2 升级到 Etcd v3，原来v2 的数据还是只能⽤ v2 的接⼝访问，v3 的接⼝创建的数据也只能访问通过 v3 的接⼝访问。

WARNING:

Environment variable ETCDCTL_API is not set; defaults to etcdctl v2. #默认使⽤V2版本

Set environment variable ETCDCTL_API=3 to use v3 API or ETCDCTL_API=2 to use v2 API. #设置API版本

wal: 存放预写式⽇志,最⼤的作⽤是记录了整个数据变化的全部历程。在etcd中，所有数据的修改在提交前，都要先写⼊到WAL中。
